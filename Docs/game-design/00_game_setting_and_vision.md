# Warzone 游戏设定与体验目标

## 1. 项目一句话定位

《Warzone》是一款后末日背景下的小规模战斗群指挥游戏。玩家不是扮演某一个英雄，也不是经营一个抽象菜单上的基地，而是在一个资源逐渐枯竭、威胁不断扩散的世界中，维持一支幸存者武装力量，带领他们进入危险地点，搜刮、救援、清理、回收，然后尽量活着撤离。

游戏的表现形式接近实时战术 RTS。玩家控制小队、火力组、车组和战斗群，而不是逐个控制每一个人。成员仍然是真实存在的：他们有武器、有生命、有压力、有装备、有死亡后果，也会在任务结束后影响基地与队伍的长期状态。玩家不需要像微操游戏那样管理每个人的站位，但每个人的状态必须能影响小队表现。

## 2. 世界观基调

游戏发生在一个失去稳定统治的后末日世界。人类社会没有完全灭绝，但原有秩序已经崩塌。道路、城镇、医院、仓库、加油站、工厂、农场、商场、学校等地点仍然存在，只是已经被感染群体、变异动物、感染植物、敌对幸存者或未知生态占据。

这个世界不应只表现为“到处都是怪物”。它还应该保留一些曾经属于人类社会的痕迹：废弃街道、临时路障、被撤离的人群痕迹、遗弃车辆、被封锁的楼房、残存的广播、旧基地、商人路线、幸存者求救信号。玩家每次进入地图时，不只是进入一个刷怪场，而是进入一个曾经真实存在、现在仍然残留信息和资源的地点。

基地承担非常重要的情绪功能。玩家不能每次进入场景都只看到敌人和消耗。基地应该是一个没有敌人的具象地点，是玩家从废墟中清理出来并逐渐修整的安全空间。它不需要成为自由摆放建筑的建设游戏，但必须能让玩家看到自己的成果：车辆停在院里，伤员在医疗区，武器摆在仓库，小队成员在休整，照明、围栏、发电机、医疗帐篷、工作台、仓储区逐渐出现。它应提供末世中稀缺的温馨感和归属感。

## 3. 核心体验

游戏最重要的体验不是“杀掉更多敌人”，而是“在有限资源和持续风险下做出可复盘的行动决策”。

一次任务中，玩家会面对这样的选择：

- 现在完成主目标后撤离，还是继续深入高价值区域？
- 是否为了回收一辆载具或一批遗失装备再打一场？
- 是否让一个受伤小队继续战斗，还是让他们撤回安全点？
- 是否夜间行动，承受更强敌人与更差视野，换取特殊材料和更高收益？
- 是否携带更多人和装备，提升战斗力，但增加消耗和撤离压力？
- 是否投入稀缺重武器清理强敌，还是保留弹药应对后续更危险区域？

这些选择应由玩家主动承担。游戏可以提供信息、提示风险、允许战术暂停，但不应把正确答案直接告诉玩家。

## 4. 失败与恢复

游戏不追求“一次失败直接结束”。成员会死，装备会丢，车辆会遗弃，任务会失败，基地会缺资源，前哨可能失去功能；但只要玩家的幸存者体系还能维持，故事就可以继续。

真正接近终局失败的情况是基地无法继续存在：食物、医疗、燃料、基础物资或可行动人员耗尽，玩家无法再组织有效行动。除此之外，战役应允许局部失败。关键人物死亡后，故事应通过物证、地点、道具、记录、其他人物或替代路线继续推进。主线不应因为一个 NPC 意外死亡而硬断。

失败不是免费试错。任务失败后，装备可能留在地图中，但会随着时间被敌人拖走、被环境破坏、被其他幸存者搜刮或逐渐散失。玩家可以返回旧地图尝试回收，但这本身是一场新的行动，仍有代价。

## 5. 战斗气质

战斗应该有致命感，但不应靠纯数值堆压玩家。参考方向是《叙利亚战争》式的战术压迫，以及 WARNO/红龙式的可读性。普通单位可以脆弱，重武器可以危险，掩体、视野、烟雾、夜战、撤离路线和敌人补充都应有意义。

高难度不应主要表现为敌人血量更厚、资源更少、玩家更憋屈。更好的难度来自更聪明的敌人、更危险的视野与夜战、更严厉的失误代价、更复杂的敌人组合和更高的决策压力。理想状态是：如果玩家侦察充分、路线合理、装备选择正确、撤离时机到位，那么高难度下也不会感到被系统恶意惩罚；如果玩家贸然深入、无视撤离、错误携装，则损失应当清楚且可复盘。

## 6. 开发原则

本项目是个人兴趣项目，不应一开始追求完整开放世界、复杂基地建造、完整敌人图鉴、全部武器、全剧情、自由地图编辑器和深度载具系统同时成立。

正确路线是先做一个持续可运行的 Demo/Sandbox 平台。这个 Demo 不是公开发布版本，也不代表最终内容量，而是一个可以快速接入、运行、观察和回归测试的验证场。每当小队移动、掩体、烟雾、夜战、任务结算、基地资源、地图重进等功能接入后，都应该能在 Demo 中直接运行，而不是等完整战役做完才知道是否好玩。
