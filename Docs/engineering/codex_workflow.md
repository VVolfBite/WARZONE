# Warzone 与 Codex 协作规则

## 1. 角色分工

用户负责提供当前代码、问题和 Codex 的执行回复。设计意图、架构边界和游戏方案由人工讨论确定，不交给 Codex 自由发散。

ChatGPT 负责阅读代码和 Codex 回复，用白话说明当前情况、风险和下一步，并给出可直接复制给 Codex 的模块级提示词。

Codex 负责执行代码改造、补测试、跑验证和报告结果。Codex 不负责重新解释游戏设计，不负责擅自改变架构目标。

## 2. 提示词跨度

Codex 提示词不应拆得过小。一个模块级功能通常最多两份提示词：

第一份用于打开接口、整理结构、完成最小可运行接入。

第二份用于补齐细节、验证、清理边界、补测试。

之后再进入新模块，或做前一模块的收尾。

## 3. Debug 提示词规则

Debug 类提示词必须在阅读代码后生成，不能只让 Codex 猜。提示词需要覆盖：

- 可能出问题的模块；
- 需要添加的 debug 日志；
- 应检查的状态流转；
- 应运行的测试；
- 可能的依赖或序列化问题；
- 修复后必须保留或新增的回归测试。

## 4. 文档规则

设计文档由 ChatGPT 直接写好，用户放到对应位置。Codex 可以按文档执行代码改造，但不应主导设计文档。

## 5. 当前第一阶段目标

第一阶段 Codex 不实现复杂玩法。它只做结构改造：目录、asmdef、依赖规则、架构测试、Core/Content/Combat/Campaign/Application/Runtime/Sandbox/Editor/Tests 的最小骨架，并迁移旧代码中可保留部分。
